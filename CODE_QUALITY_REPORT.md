## 1. Duplicate Code Patterns

### 1.1 Numeric Guard Functions (305+ occurrences)

**Pattern:** `Number()`, `isNaN()`, `isFinite()` scattered across codebase

**Locations with highest duplication:**
- `lib/utils.ts` - `isNaN()` checks
- `lib/analytics/var.ts` - `toNumber()`, `Number.isFinite()` 
- `lib/tick-calculations.ts` - `Number()` conversions
- `server/teams.ts` - Multiple `Number(trade.pnl)` calls
- `lib/redis-cache.ts` - Number parsing
- `lib/query-optimizer.ts` - Number parsing (duplicate of redis-cache)

**Recommendation:** Consolidate into shared utilities:
```typescript
// lib/utils.ts - add:
export function toFiniteNumber(value: unknown): number {
  const num = typeof value === 'number' ? value : Number(value)
  return Number.isFinite(num) ? num : 0
}

export function safeDivide(numerator: number, denominator: number): number {
  return denominator === 0 ? 0 : numerator / denominator
}
```

### 1.2 Date Parsing (465+ occurrences)

**Pattern:** `new Date()` called repeatedly without safe parsing

**Locations:**
- `lib/date-utils.ts` - Safe parsing functions exist but underused
- `lib/data-types.ts` - Date normalization
- `lib/behavior-insights.ts` - Date parsing
- `lib/analytics/metrics-v1.ts` - Date conversions
- 50+ other files

**Current utilities in `lib/date-utils.ts`:**
- `formatTimestamp()` - Normalizes ISO timestamps
- `normalizeToUtcTimestamp()` - Safe UTC parsing
- `toValidDate()` - Null-safe date parsing (NOT USED elsewhere)

**Recommendation:** Use existing `toValidDate()` from lib/date-utils.ts more widely

### 1.3 Console.error Logging (330+ occurrences)

**Pattern:** Inline try/catch with console.error scattered everywhere

**Example duplicates:**
```typescript

```

**Recommendation:** Use centralized logger from `lib/logger.ts`
